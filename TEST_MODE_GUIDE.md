# 🧪 測試模式使用指南

## 進入測試模式

### 觸發方式：
1. **連續點擊 10 次**左上角的"東京輕井澤親子之旅2026/1/24 - 1/29"標題卡片
   - 每次點擊會顯示進度提示 (e.g., "點擊 3/10 🩷")
   
2. **點擊右上角的鎖定按鈕**（當計數達到 10 次時）
   - 當點擊計數達到 10 次後，鎖定按鈕會變成粉紅色的 Key 圖示並閃爍
   - 此時點擊鎖定按鈕即可進入測試模式

### 重置計數：
- 如果中途進行以下操作，計數會被重置為 0，需要重新開始：
  - 點擊任何 **Tab 按鈕**（行程、指南、AI、商家、連結）
  - **滑動** 頁面切換日期
  - 點擊其他功能按鈕（除了標題和鎖定按鈕）

## 測試模式功能

### 1. 📅 日期與時間調整
- **自由設定日期**：修改當前日期以觀察不同日期下的行程資訊
- **自由設定時間**：修改當前時間以查看時間敏感的內容變化
- 時間戳記會實時更新，顯示當前設定的日期時間

### 2. 🗺️ 位置座標調整
- **緯度 (Latitude)**：設定 N(北)南座標
- **經度 (Longitude)**：設定 E(東)西座標
- **預設座標**：輕井澤 (35.4437, 138.3919)
- 修改座標後，天氣和位置資訊會**同步更新**

### 3. ☁️ 天氣覆蓋功能
測試模式可以強制修改任何日期的天氣，不受實際 API 數據限制

#### 可選的天氣代碼：
- **0** - 晴朗 ☀️
- **1, 2, 3** - 多雲 ☁️
- **45, 48** - 有霧 🌫️
- **51, 55** - 下雨 🌧️
- **71, 75** - 下雪 ❄️
- **95** - 雷雨 ⛈️
- **自動** - 使用實際天氣預報數據

#### 應用天氣的位置：
- **總覽天氣**：影響首頁显示的天氣資訊和背景特效
- **Day 1 ~ Day 6**：分別影響每日行程頁面的天氣顯示

## 退出測試模式

在測試模式面板的底部點擊 **「退出測試模式」** 紅色按鈕即可退出。

## 實用場景

### 場景 1：驗證不同天氣的 UI 效果
- 設定天氣代碼為 71（下雪），觀察背景特效變化
- 驗證各種天氣條件下的視覺表現和穿搭建議

### 場景 2：測試時間敏感功能
- 修改日期時間觀察倒計時、進度指示的變化
- 檢驗旅程進度顯示（旅程前/中/後的不同內容）

### 場景 3：測試位置變更功能
- 在不同座標間切換，驗證位置信息和天氣更新是否同步
- 確保 GPS 定位模擬工作正常

## 技術細節

- 測試模式的所有設定值存儲在 React State 中，**不會持久化**
- 刷新頁面後測試模式會自動關閉，所有設定會重置
- 測試模式不影響已保存的使用者數據（檢查清單、聊天記錄等）

**提示**：這個測試模式設計用於開發和 QA 測試，確保應用在各種極端情況下的可靠性。
